## UX/UI Enhancement Ideas

### 1. Step Editor UI Redesign üìù
**Priority**: High
**Status**: ‚è≠Ô∏è Not Started

**Goal**: Redesign the step editing interface for better usability and faster workflow.

**Layout Requirements**:
- [ ] **Step Number Field**: Small, compact input box positioned in the top-right corner of the step editor, next to the delete button
- [ ] **Description Field**: Large text area positioned in the top-left, taking up the majority of the horizontal space
- [ ] **Notes Field**: Full-width text area positioned below Description and Step Number, with adequate height for multi-line content
- [ ] **Screenshots Section**: Positioned below Notes field, with clear visual separation

**Screenshot Management**:
- [ ] **Screenshot Naming**: Add an editable text field for each screenshot to assign a custom name (e.g., "order_creation_screen", "compliance_check_result")
- [ ] **Screenshot References in Notes**: Enable users to reference screenshots in the Notes field using `@screenshot_name` syntax (e.g., "@order_creation_screen shows the validation")
- [ ] **Drag & Drop Zone**: Make the entire step editor area a drop zone for screenshots - users can drag and drop images anywhere within the step editor, not just in the designated upload area
- [ ] **Visual Feedback**: Show clear visual indicators when dragging files over the step editor area

**Benefits**:
- Faster data entry workflow
- More intuitive screenshot management
- Better organization with named screenshots
- Ability to reference screenshots in notes for better documentation
